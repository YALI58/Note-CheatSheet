# Redisson分布式锁与幂等性控制定义速查表

## 一、核心概念定义

### 1. 分布式锁 (Distributed Lock)

| 术语 | 定义 | 关键特性 |
|------|------|----------|
| **分布式锁** | 在分布式系统中，协调多个节点对共享资源进行互斥访问的机制 | 互斥性、超时释放、可重入 |
| **互斥性** | 同一时刻只能有一个线程/进程持有锁 | 通过Redis原子命令保证 |
| **可重入性** | 同一个线程可以多次获取同一把锁 | 通过计数器实现 |
| **锁超时** | 防止死锁，自动释放锁的时间机制 | 支持看门狗自动续期 |
| **看门狗** | 自动续期机制，防止业务未完成锁过期 | 默认每10秒续期一次 |

### 2. Redisson锁类型定义

| 锁类型 | 类名 | 定义 | 适用场景 |
|--------|------|------|----------|
| **可重入锁** | `RLock` | 同一个线程可多次加锁 | 大多数业务场景 |
| **公平锁** | `RFairLock` | 按请求顺序分配锁 | 任务排队处理 |
| **读写锁** | `RReadWriteLock` | 读写分离，读读不互斥 | 读多写少场景 |
| **联锁** | `RMultiLock` | 同时获取多个锁 | 需要锁定多个资源 |
| **红锁** | `RedissonRedLock` | 在多个Redis节点上获取锁 | 高可用场景 |

### 3. 幂等性控制定义

| 术语 | 定义 | 实现方式 |
|------|------|----------|
| **幂等性** | 同一操作执行多次与执行一次效果相同 | 通过唯一标识去重 |
| **幂等令牌** | 标识请求唯一性的token | 每次请求携带唯一值 |
| **去重窗口** | 幂等控制的有效时间范围 | 设置Redis key过期时间 |
| **业务锁** | 防止并发重复提交的锁 | 分布式锁+状态标记 |

## 五、Lua脚本定义

### 1. 加锁脚本

```lua
-- 加锁脚本定义
local key = KEYS[1]                 -- 锁的key
local threadId = ARGV[1]            -- 线程ID
local ttl = tonumber(ARGV[2])        -- 过期时间

-- 如果锁不存在
if (redis.call('exists', key) == 0) then
    redis.call('hset', key, threadId, '1')
    redis.call('pexpire', key, ttl)
    return nil
end

-- 如果锁被当前线程持有
if (redis.call('hexists', key, threadId) == 1) then
    redis.call('hincrby', key, threadId, '1')
    redis.call('pexpire', key, ttl)
    return nil
end

-- 返回剩余过期时间
return redis.call('pttl', key)
```

### 2. 解锁脚本

```lua
-- 解锁脚本定义
local key = KEYS[1]
local threadId = ARGV[1]

-- 如果不是当前线程持有
if (redis.call('hexists', key, threadId) == 0) then
    return nil
end

-- 递减计数器
local counter = redis.call('hincrby', key, threadId, -1)

-- 如果计数器大于0，刷新过期时间
if (counter > 0) then
    redis.call('pexpire', key, ARGV[2])
    return 0
else
    -- 计数器为0，删除锁
    redis.call('del', key)
    return 1
end
```

```java
        // 业务逻辑
```

### 2. 幂等性切面实现

```java
@Aspect
@Component
public class IdempotentAspect {
    
    @Autowired
    private RedissonClient redissonClient;
    
    @Autowired
    private HttpServletRequest request;
    
    @Around("@annotation(idempotent)")
    public Object around(ProceedingJoinPoint joinPoint, Idempotent idempotent) throws Throwable {
        String key = generateKey(joinPoint, idempotent);
        RLock lock = redissonClient.getLock("idempotent:lock:" + key);
        
        // 尝试获取分布式锁
        if (!lock.tryLock(0, idempotent.expireTime(), TimeUnit.SECONDS)) {
            throw new RuntimeException(idempotent.message());
        }
        
        try {
            // 检查是否已处理过
            RBucket<Object> bucket = redissonClient.getBucket("idempotent:processed:" + key);
            if (bucket.isExists()) {
                throw new RuntimeException("请求已处理，请勿重复提交");
            }
            
            // 执行原方法
            Object result = joinPoint.proceed();
            
            // 标记为已处理
            bucket.set("processed", idempotent.expireTime(), TimeUnit.SECONDS);
            
            return result;
        } finally {
            // 释放锁
            if (lock.isLocked() && lock.isHeldByCurrentThread()) {
                lock.unlock();
            }
        }
    }
    
    private String generateKey(ProceedingJoinPoint joinPoint, Idempotent idempotent) {
        switch (idempotent.keyType()) {
            case TOKEN:
                String token = request.getHeader("Authorization");
                if (token == null) {
                    token = request.getParameter("token");
                }
                return idempotent.keyPrefix() + ":" + token;
                
            case USER_ID:
                // 从SecurityContext或Session获取用户ID
                String userId = getCurrentUserId();
                return idempotent.keyPrefix() + ":" + userId;
                
            case REQUEST_URL:
                String url = request.getRequestURI();
                String queryString = request.getQueryString();
                return idempotent.keyPrefix() + ":" + url + ":" + queryString;
                
            case CUSTOM:
                // 使用SpringEL表达式解析
                return parseExpression(joinPoint, idempotent.keyExpression());
                
            default:
                return idempotent.keyPrefix() + ":" + UUID.randomUUID().toString();
        }
    }
    
    private String getCurrentUserId() {
        // 实现获取当前用户ID的逻辑
        return "user123";
    }
    
    private String parseExpression(ProceedingJoinPoint joinPoint, String expression) {
        // 实现SpringEL表达式解析
        return "customKey";
    }
}
```

### 1. 分布式锁工具类

```java
@Component
public class RedisLockUtil {
    
    @Autowired
    private RedissonClient redissonClient;
    
    /**
     * 带回调的执行方法
     */
    public <T> T executeWithLock(String key, long waitTime, long leaseTime, 
                                 TimeUnit unit, Supplier<T> supplier) {
        RLock lock = redissonClient.getLock(key);
        try {
            if (lock.tryLock(waitTime, leaseTime, unit)) {
                try {
                    return supplier.get();
                } finally {
                    if (lock.isLocked() && lock.isHeldByCurrentThread()) {
                        lock.unlock();
                    }
                }
            }
            throw new RuntimeException("获取锁失败");
        } catch (InterruptedException e) {
            Thread.currentThread().interrupt();
            throw new RuntimeException("线程被中断", e);
        }
    }
    
    /**
     * 执行无返回值的任务
     */
    public void runWithLock(String key, long waitTime, long leaseTime,
                           TimeUnit unit, Runnable task) {
        executeWithLock(key, waitTime, leaseTime, unit, () -> {
            task.run();
            return null;
        });
    }
}
```

```java
public class IdempotentUtil {

     * 生成幂等令牌
     */
    public String generateIdempotentToken(String businessType) {

        RBucket<String> bucket = redissonClient.getBucket("idempotent:token:" + token);
        bucket.set("pending", 30, TimeUnit.MINUTES);
        return token;
    }

     * 校验幂等令牌
     */
    public boolean validateIdempotentToken(String token) {
        String key = "idempotent:token:" + token;
        RBucket<String> bucket = redissonClient.getBucket(key);
        
        // CAS操作，确保幂等性
        return bucket.compareAndSet("pending", "processed");
    }
    
    /**
     * 基于业务ID的幂等校验
     */
    public boolean checkIdempotent(String businessId, long expireTime) {
        String key = "idempotent:business:" + businessId;
        RLock lock = redissonClient.getLock(key + ":lock");
        
        try {
            if (lock.tryLock(0, expireTime, TimeUnit.SECONDS)) {
                try {
                    RBucket<String> bucket = redissonClient.getBucket(key);
                    if (bucket.isExists()) {
                        return false;
                    }
                    bucket.set("processed", expireTime, TimeUnit.SECONDS);
                    return true;
                } finally {
                    lock.unlock();
                }
            }
            return false;
        } catch (InterruptedException e) {
            Thread.currentThread().interrupt();
            return false;
        }
    }
```

### 3. 常见问题处理

```java
// 锁超时导致业务未完成的问题
// 解决方案：使用看门狗机制
public void safeLock(String key) {
    RLock lock = redissonClient.getLock("lock:" + key);
    
    // 不指定leaseTime，启用看门狗
    lock.lock();
    try {
        // 长时间业务
    } finally {
        // 确保只释放当前线程持有的锁
        if (lock.isLocked() && lock.isHeldByCurrentThread()) {
            lock.unlock();
        }
    }
}

// 防止锁被其他线程释放
public void safeUnlock(RLock lock) {
    if (lock.isLocked() && lock.isHeldByCurrentThread()) {
        lock.unlock();
    }
}
```

## 七、性能优化建议
