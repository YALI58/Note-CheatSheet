## Java函数式接口详解

### 什么是函数式接口？

**函数式接口**（Functional Interface）是指**只有一个抽象方法**的接口。Java 8引入这一概念，主要用于支持Lambda表达式。

### 基本特性

1. **只能有一个抽象方法**
2. 可以有多个默认方法和静态方法
3. 可以使用`@FunctionalInterface`注解标记（可选，但推荐）

### 基本示例

```java
// 正确的函数式接口
@FunctionalInterface
interface Calculator {
    int calculate(int a, int b);  // 只有一个抽象方法
    
    // 可以有默认方法
    default void printResult(int result) {
        System.out.println("结果: " + result);
    }
    
    // 可以有静态方法
    static void showInfo() {
        System.out.println("这是一个计算器接口");
    }
}

// 使用Lambda表达式实现
public class Test {
    public static void main(String[] args) {
        // Lambda实现加法
        Calculator add = (a, b) -> a + b;
        System.out.println(add.calculate(5, 3));  // 输出: 8
        
        // Lambda实现乘法
        Calculator multiply = (a, b) -> a * b;
        System.out.println(multiply.calculate(5, 3));  // 输出: 15
        
        // 调用默认方法
        multiply.printResult(15);  // 输出: 结果: 15
    }
}
```

### 常见的错误示例

```java
// ❌ 错误：多个抽象方法
@FunctionalInterface
interface WrongInterface {
    void method1();
    void method2();  // 编译错误！
}

// ❌ 错误：没有抽象方法
@FunctionalInterface
interface NoMethod {  // 编译错误！
    default void method() {}
}

// ✅ 正确：继承多个接口，但只有一个抽象方法
interface A {
    void method1();
}

interface B {
    default void method2() {}
}

@FunctionalInterface
interface C extends A, B {
    // 只有从A继承的一个抽象方法
}
```

## Java内置的函数式接口

Java 8在`java.util.function`包中提供了43个标准函数式接口，主要分为四大类：

### 1. **Consumer（消费者）**
接收一个参数，不返回结果

```java
@FunctionalInterface
public interface Consumer<T> {
    void accept(T t);
}

// 使用示例
Consumer<String> printer = s -> System.out.println(s);
printer.accept("Hello");  // 输出: Hello

// 链式调用
Consumer<String> first = s -> System.out.print("开始:");
Consumer<String> second = s -> System.out.println(s);
Consumer<String> chain = first.andThen(second);
chain.accept("Java");  // 输出: 开始:Java

// 特殊变体：BiConsumer（两个参数）
BiConsumer<String, Integer> printInfo = (name, age) -> 
    System.out.println(name + "今年" + age + "岁");
printInfo.accept("张三", 25);
```

### 2. **Supplier（提供者）**
不接受参数，返回一个结果

```java
@FunctionalInterface
public interface Supplier<T> {
    T get();
}

// 使用示例
Supplier<String> stringSupplier = () -> "Hello World";
System.out.println(stringSupplier.get());  // 输出: Hello World

// 随机数生成器
Supplier<Double> randomSupplier = () -> Math.random();
System.out.println(randomSupplier.get());

// 工厂模式示例
Supplier<Person> personFactory = () -> new Person("默认姓名", 18);
Person p = personFactory.get();
```

### 3. **Function（函数）**
接收一个参数，返回一个结果

```java
@FunctionalInterface
public interface Function<T, R> {
    R apply(T t);
}

// 使用示例
Function<String, Integer> strLength = s -> s.length();
System.out.println(strLength.apply("Hello"));  // 输出: 5

// 链式调用：先转大写，再获取长度
Function<String, String> toUpper = s -> s.toUpperCase();
Function<String, Integer> upperLength = toUpper.andThen(strLength);
System.out.println(upperLength.apply("hello"));  // 输出: 5

// compose方法：先执行参数，再执行调用者
Function<Integer, Integer> multiply2 = x -> x * 2;
Function<Integer, Integer> add3 = x -> x + 3;
Function<Integer, Integer> add3ThenMultiply2 = multiply2.compose(add3);
System.out.println(add3ThenMultiply2.apply(5));  // (5+3)*2 = 16

// 特殊变体：BiFunction（两个参数，一个结果）
BiFunction<Integer, Integer, String> sumToString = (a, b) -> 
    "和是: " + (a + b);
System.out.println(sumToString.apply(5, 3));  // 输出: 和是: 8
```

### 4. **Predicate（断言）**
接收一个参数，返回boolean值

```java
@FunctionalInterface
public interface Predicate<T> {
    boolean test(T t);
}

// 使用示例
Predicate<String> isEmpty = s -> s == null || s.isEmpty();
System.out.println(isEmpty.test(""));  // 输出: true
System.out.println(isEmpty.test("abc"));  // 输出: false

// 逻辑操作：and, or, negate
Predicate<Integer> greaterThan10 = x -> x > 10;
Predicate<Integer> lessThan20 = x -> x < 20;

// and: 10 < x < 20
Predicate<Integer> between10And20 = greaterThan10.and(lessThan20);
System.out.println(between10And20.test(15));  // 输出: true

// or: x < 10 或 x > 20
Predicate<Integer> outside10To20 = greaterThan10.negate().or(lessThan20.negate());
// 简化写法
Predicate<Integer> outside = greaterThan10.negate().or(lessThan20.negate());

// 静态方法isEqual
Predicate<String> isHello = Predicate.isEqual("Hello");
System.out.println(isHello.test("Hello"));  // 输出: true

// 特殊变体：BiPredicate（两个参数）
BiPredicate<String, Integer> checkLength = (s, len) -> s.length() == len;
System.out.println(checkLength.test("Java", 4));  // 输出: true
```

## 其他重要的函数式接口

### Operator（操作符）
Function的特殊情况，参数和返回值类型相同

```java
// UnaryOperator：一元操作符，继承Function<T, T>
UnaryOperator<String> toUpperCase = s -> s.toUpperCase();
System.out.println(toUpperCase.apply("hello"));  // 输出: HELLO

// BinaryOperator：二元操作符，继承BiFunction<T, T, T>
BinaryOperator<Integer> add = (a, b) -> a + b;
System.out.println(add.apply(5, 3));  // 输出: 8

// 提供两个静态方法
BinaryOperator<Integer> maxBy = BinaryOperator.maxBy(Integer::compare);
System.out.println(maxBy.apply(10, 20));  // 输出: 20

BinaryOperator<Integer> minBy = BinaryOperator.minBy(Integer::compare);
System.out.println(minBy.apply(10, 20));  // 输出: 10
```

### 原始类型特化
为了避免自动装箱的性能开销，为基本类型提供了专用接口

```java
// 避免装箱的示例
IntSupplier intSupplier = () -> 10;  // 返回int，不是Integer
IntConsumer intConsumer = x -> System.out.println(x);
IntFunction<String> intToString = x -> "数字: " + x;
IntPredicate evenNumber = x -> x % 2 == 0;

// 性能对比
// 使用装箱类型（较慢）
Predicate<Integer> boxedEven = x -> x % 2 == 0;

// 使用原始类型（更快）
IntPredicate primitiveEven = x -> x % 2 == 0;

// 测试10万个数的性能差异
long start = System.nanoTime();
for (int i = 0; i < 100000; i++) {
    boxedEven.test(i);  // 需要装箱
}
long boxedTime = System.nanoTime() - start;

start = System.nanoTime();
for (int i = 0; i < 100000; i++) {
    primitiveEven.test(i);  // 无需装箱
}
long primitiveTime = System.nanoTime() - start;

System.out.println("装箱版本时间: " + boxedTime);
System.out.println("原始类型版本时间: " + primitiveTime);
// 原始类型版本通常快2-5倍
```

## 实际应用示例

### 1. **结合Stream API使用**

```java
import java.util.Arrays;
import java.util.List;
import java.util.stream.Collectors;

public class StreamExample {
    public static void main(String[] args) {
        List<String> names = Arrays.asList("Alice", "Bob", "Charlie", "David");
        
        // 使用Predicate过滤
        List<String> longNames = names.stream()
            .filter(name -> name.length() > 4)  // Predicate
            .collect(Collectors.toList());
        System.out.println("长名字: " + longNames);  // [Alice, Charlie]
        
        // 使用Function转换
        List<Integer> nameLengths = names.stream()
            .map(String::length)  // Function
            .collect(Collectors.toList());
        System.out.println("名字长度: " + nameLengths);  // [5, 3, 7, 5]
        
        // 使用Consumer遍历
        names.forEach(System.out::println);  // Consumer
        
        // 使用Supplier生成流
        List<Double> randomNumbers = Stream.generate(Math::random)  // Supplier
            .limit(5)
            .collect(Collectors.toList());
        System.out.println("随机数: " + randomNumbers);
    }
}
```

### 2. **方法引用**

```java
import java.util.function.*;

public class MethodReferenceExample {
    public static void main(String[] args) {
        // 静态方法引用
        Function<String, Integer> parseInt = Integer::parseInt;
        // 等同于: s -> Integer.parseInt(s)
        
        // 实例方法引用（特定对象）
        List<String> list = Arrays.asList("a", "b", "c");
        Consumer<String> print = System.out::println;
        // 等同于: s -> System.out.println(s)
        
        // 实例方法引用（任意对象）
        Function<String, String> toUpperCase = String::toUpperCase;
        // 等同于: s -> s.toUpperCase()
        
        // 构造方法引用
        Supplier<List<String>> listSupplier = ArrayList::new;
        // 等同于: () -> new ArrayList<>()
        
        Function<String, Person> personCreator = Person::new;
        // 等同于: name -> new Person(name)
    }
}
```

### 3. **策略模式实现**

```java
import java.util.function.*;

// 使用函数式接口简化策略模式
class PaymentStrategy {
    private final Function<Double, String> paymentProcessor;
    
    public PaymentStrategy(Function<Double, String> paymentProcessor) {
        this.paymentProcessor = paymentProcessor;
    }
    
    public String processPayment(double amount) {
        return paymentProcessor.apply(amount);
    }
    
    // 预定义的支付策略
    public static final Function<Double, String> CREDIT_CARD = 
        amount -> String.format("信用卡支付: %.2f元", amount);
    
    public static final Function<Double, String> ALIPAY = 
        amount -> String.format("支付宝支付: %.2f元", amount);
    
    public static final Function<Double, String> WECHAT = 
        amount -> String.format("微信支付: %.2f元", amount);
}

// 使用
public class StrategyDemo {
    public static void main(String[] args) {
        PaymentStrategy creditCard = new PaymentStrategy(PaymentStrategy.CREDIT_CARD);
        System.out.println(creditCard.processPayment(100.50));
        
        // 动态创建新策略
        PaymentStrategy discount = new PaymentStrategy(
            amount -> String.format("折扣后: %.2f元", amount * 0.8)
        );
        System.out.println(discount.processPayment(100));
    }
}
```

### 4. **回调函数**

```java
import java.util.function.*;

class AsyncTask {
    public void execute(Runnable task, Consumer<Exception> errorHandler) {
        new Thread(() -> {
            try {
                task.run();
            } catch (Exception e) {
                errorHandler.accept(e);
            }
        }).start();
    }
    
    public <T> void executeWithResult(Supplier<T> task, 
                                      Consumer<T> successHandler,
                                      Consumer<Exception> errorHandler) {
        new Thread(() -> {
            try {
                T result = task.get();
                successHandler.accept(result);
            } catch (Exception e) {
                errorHandler.accept(e);
            }
        }).start();
    }
}

// 使用
public class CallbackDemo {
    public static void main(String[] args) {
        AsyncTask task = new AsyncTask();
        
        task.execute(
            () -> System.out.println("任务执行中..."),
            error -> System.err.println("错误: " + error.getMessage())
        );
        
        task.executeWithResult(
            () -> "计算结果",
            result -> System.out.println("成功: " + result),
            error -> System.err.println("失败: " + error.getMessage())
        );
    }
}
```

## 最佳实践与注意事项

### 1. **优先使用标准函数式接口**
```java
// ❌ 避免自定义
interface StringProcessor {
    String process(String s);
}

// ✅ 使用标准接口
Function<String, String> processor = s -> s.trim();
```

### 2. **使用@FunctionalInterface注解**
```java
// 明确这是一个函数式接口，编译器会检查
@FunctionalInterface
interface MyInterface {
    void doSomething();
}
```

### 3. **注意装箱性能**
```java
// 大量数据时避免装箱
// ❌ 慢
IntStream.range(0, 1000000)
    .mapToObj(i -> i)  // 装箱
    .filter(i -> i % 2 == 0)  // 再次装箱？
    .count();

// ✅ 快
IntStream.range(0, 1000000)
    .filter(i -> i % 2 == 0)
    .count();
```

### 4. **Lambda表达式的变量捕获**
```java
public class VariableCapture {
    public void test() {
        int localVar = 10;  // 必须是 effectively final
        // localVar = 20;  // 如果修改这里，下面的lambda会编译错误
        
        Runnable r = () -> {
            System.out.println(localVar);  // 捕获局部变量
            // localVar = 30;  // 编译错误：不能修改捕获的变量
        };
    }
}
```

### 5. **异常处理**
```java
// 函数式接口的标准实现不抛出受检异常
Function<String, Integer> risky = s -> {
    try {
        return Integer.parseInt(s);
    } catch (NumberFormatException e) {
        throw new RuntimeException(e);  // 包装为非受检异常
    }
};

// 或者创建自己的函数式接口支持异常
@FunctionalInterface
interface ThrowingFunction<T, R> {
    R apply(T t) throws Exception;
}
```

函数式接口是Java函数式编程的基础，掌握它们可以让你写出更简洁、更灵活的代码。
