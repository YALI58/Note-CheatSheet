# 🚀 IDEA调试技巧速查表
*一眼找到你需要的调试技能*

---

## 🟢 基础操作篇

| 你要做什么 | 按什么键 | 一句话说明 |
|:---|:---:|:---|
| **下一步**（不进入方法） | `F8` | 跳过方法内部，只看当前行 |
| **钻进去**（进入方法） | `F7` | 想看方法里怎么写的就用它 |
| **强制钻进去** | `Alt+Shift+F7` | 连JDK源码都不放过 |
| **跳出来**（回到上层） | `Shift+F8` | 看完方法内部了，出来吧 |
| **继续运行** | `F9` | 跑到下一个断点去 |
| **停止调试** | `Ctrl+F2` | 不玩了 |
| **计算表达式** | `Alt+F8` | 现场执行代码/改值 |

## 🔵 断点花样玩法

| 断点类型 | 长啥样 | 什么时候用 | 怎么设置 |
|:---|:---:|:---|:---|
| **普通断点** | ● 红点 | 就想停在这一行 | 点行号后面 |
| **条件断点** | ● 带? | 循环第500次才停 | 右键断点 → Condition |
| **日志断点** | ● 带✓ | 只想打印日志，不想停 | 右键 → 取消Suspend → 勾Log |
| **方法断点** | ◇ 菱形 | 想看这个方法何时被调用 | 点方法定义行 |
| **异常断点** | ❗ 闪电 | NPE找不到哪抛的 | `Ctrl+Shift+F8` → + → 异常类 |
| **字段断点** | 👁️ 眼睛 | 想知道变量何时被改 | 点字段定义行 |

## 🟠 高级骚操作

| 技巧 | 快捷键/操作 | 装逼指数 | 实用场景 |
|:---|:---:|:---:|:---|
| **强制返回** | 右键栈帧 → Force Return | ⭐⭐⭐⭐⭐ | 方法不想执行完了，提前溜 |
| **抛异常** | 右键栈帧 → Throw Exception | ⭐⭐⭐⭐ | 测试异常处理逻辑 |
| **改值** | 变量上按 `F2` | ⭐⭐⭐⭐ | 把null改成"正常值"继续跑 |
| **回退栈帧** | 点击栈帧 → Drop Frame | ⭐⭐⭐⭐⭐ | 进错方法了，退回去重来 |
| **Stream调试** | Debug时点击🔍按钮 | ⭐⭐⭐⭐⭐ | 看map/filter中间结果 |
| **查看调用链** | 点击Frames层数 | ⭐⭐⭐ | 回溯方法怎么来的 |

## 🟣 实战场景速查

### 场景1：循环太大，只想看某一次
👉 **用条件断点**：右键断点 → 输入 `i == 500`

### 场景2：不想停程序，只想看日志
👉 **用日志断点**：右键断点 → 取消Suspend → 勾Log

### 场景3：空指针找不到源头
👉 **用异常断点**：`Ctrl+Shift+F8` → + → `NullPointerException`

### 场景4：变量值被改了，不知道谁干的
👉 **用字段断点**：在字段定义行打点

### 场景5：Stream操作看不懂中间结果
👉 **用Stream Trace**：断点停在Stream行 → 点击🔍按钮

### 场景6：线上出bug本地复现不了
👉 **远程调试**：Run → Edit Configurations → + → Remote

## 📌 记住这几条就够了

| 问题 | 解决方案 |
|:---|:---|
| 不想一直按F8 | **条件断点** |

| 不知道异常哪来的 | **异常断点** |
| 不知道变量谁改的 | **字段断点** |

| 想看Stream中间过程 | **Stream Trace** |
